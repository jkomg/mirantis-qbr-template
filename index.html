TARP

Tactical Account Reporting Tool

Two account-reporting tools that run locally against data on this machine. One builds a Quarterly Business Review deck for a single account; the other reports on SLA and service performance across the whole portfolio. Each card below covers what it does, how to use it, and where its data lives.

QBR DECK BUILDER
Configurator → Deck →
Turns one account's data into an 18-slide Quarterly Business Review — exec summary, health, usage, incidents, wins, risks, roadmap, and renewal math — ready to present or export to PPTX.
HOW TO USE
Start in the Configurator: paste a Salesforce export or type values in by hand, toggle slides on/off, then open the Deck. Its Tweaks panel adjusts tone, length, and framing per meeting.
DATA & EPHEMERALITY
Your draft is saved to this browser's localStorage under the key qbr_data_draft — visible in DevTools → Application → Local Storage. Clicking Download JSON writes qbr.data.json to your browser's normal downloads folder; move that file next to QBR Template.dc.html if you want the Deck to load it automatically without localStorage.
SLA PERFORMANCE REPORT
Portfolio → Account detail →
Aggregates support-ticket history across every account into one portfolio view — severity lifecycle (what's SLA-bound vs. not), resolution time, recurring themes — then lets you click into any account for its full ticket history.
HOW TO USE
Open it directly, no setup step. Click any account row to drill in; click a column header to re-sort. Reload the page any time to re-read the latest data.
DATA & EPHEMERALITY
Reads accounts/*.json live, fresh, on every load — a read-only mount, nothing cached or written back. It has no memory between visits; update a file on disk and reload to see the change.
